# Pokedex 一覧/詳細のデータ取得戦略

## 現状

- 一覧: `pokemon` 一覧APIを取得後、表示名の日本語化のために `species` を件数分追加取得しています（N+1）。
- 詳細: `detail` / `species` に加え、進化・特性・技を画面要件に合わせて追加取得しています。

## 意図

本実装は**学習目的でクライアント集約**を採用しています。React Query のキャッシュや並列取得を理解する教材として、あえて複数クエリを組み合わせる構成にしています。

## 制約

- 一覧件数に比例して API 呼び出し数が増加します。
- ネットワーク遅延が大きい環境では、初回表示やページ追加時に待ち時間が伸びます。
- 実務では API レート制限やバックエンド負荷の面で成立しない可能性があります。

## 実務での改善候補

1. BFF/集約APIを導入し、一覧表示に必要な項目をサーバー側で整形して返す。
2. 一覧では英語名など最小情報のみ表示し、日本語名は詳細画面で取得する。
3. クライアント側の事前集約関数（usecase）を導入し、hook には集約済みモデルのみ渡す。
